# DB seed data

`packages/db` の開発用 seed は、チケット販売の代表的なイベント形状をまとめて作る。

実行コマンド:

```sh
bun run db:seed
```

内部では `packages/db/src/seed/scenarios.ts` の `seedScenarioPatterns` をもとに、以下の6パターンを作る。

- 1公演1受付1席種1料金種別
- 1公演1受付1席種2料金種別
- 1公演2受付2席種1料金種別
- 3公演 通し券あり1料金種別
- 20公演2受付5席種1料金種別
- 20公演2受付5席種2料金種別

各パターンはさらに以下の4バリエーションへ展開する。

- 整理番号 + 先着
- 整理番号 + 抽選
- 指定席 + 先着
- 指定席 + 抽選

合計で24イベントを作成する。
データは `[Seed]` から始まるイベント名と `seed-` prefix のIDを持つ。
既定の `db:seed` は同じ seed マスタを削除してから作り直す。
seed イベントに注文・チケットなどの取引データを追加している場合、外部キー制約で削除が止まることがある。

新しいパターンを追加するときは `seedScenarioPatterns` に1件足し、`packages/db/src/seed/scenarios.unit.test.ts` で期待する組み合わせ数や通し券などの重要条件を確認する。
